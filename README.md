# Chronobeats

## Introduction
Welcome to Chrono Beats, a project that was undertaken by a team of developers as part of a coding boot camp. The goal of the project was to compare and test the performance of two different database products - Postgres and MongoDB - for a music consumption application. The team built a simple frontend to mimic popular music apps and used K6 testing to perform load, spike, soak, and stress tests on both databases.


## Features

- Comparing and testing the performance of Postgres and MongoDB using K6.
- Live previews
- Developing modular components using Docker for easy testing and comparison of database solutions.
- Selecting Postgres as the optimal database solution due to its 27% faster response time and lower failure rate during all test scenarios.
- Conducting load, spike, soak, and stress tests to thoroughly assess the strengths and weaknesses of each database solution.

## Conclusion
The Chrono Beats project was a valuable learning experience for the development team, as they gained experience working with both Postgres and MongoDB and conducting performance tests. The results showed that Postgres was the best choice for the music consumption application due to its faster response time and lower failure rate. The modular components developed using Docker will allow for easy testing and comparison of different database solutions in the future
